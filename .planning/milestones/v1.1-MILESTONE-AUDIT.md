## Incident Note — `--delete-data` Mishap (2026-03-17)

A production data wipe occurred when `spacetime publish --delete-data` was run manually, bypassing `make publish`. Full recovery was completed same day (0 failures) using `recovery/restore.js`, `recovery/restore-history.js`, and `recovery/compute-tiers.js`.

**Recovery artifacts preserved in:** `recovery/INCIDENT.md`

**Lessons applied to v1.1:**
- `make publish` is the only safe publish path — never raw `spacetime publish`
- Export CSVs before any schema-breaking deploy
- Restore reducers are idempotent and remain in the server as a safety net
