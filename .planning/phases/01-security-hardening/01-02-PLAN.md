    **New reducer 1 — get_my_recovery_code:**
    ```rust
    #[reducer]
    pub fn get_my_recovery_code(ctx: &ReducerContext) -> Result<(), String> {
        let _player = get_player(ctx)?;
        let record = ctx.db.recovery_keys()
            .iter()
            .find(|k| k.owner == ctx.sender())
            .ok_or("No recovery key found")?;
        match ctx.db.recovery_code_results().owner().find(ctx.sender()) {
            Some(existing) => {
                ctx.db.recovery_code_results().owner().update(RecoveryCodeResult {
                    owner: ctx.sender(),
                    code: record.code.clone(),
                });
            }
            None => {
                ctx.db.recovery_code_results().insert(RecoveryCodeResult {
                    owner: ctx.sender(),
                    code: record.code.clone(),
                });
            }
        }
        Ok(())
    }
    ```
    Place after `RecoveryCodeResult` struct.

    **New private table 2 — TransferCodeResult:**
    ```rust
    #[table(accessor = transfer_code_results)]
    pub struct TransferCodeResult {
        #[primary_key]
        pub owner: Identity,
        pub code: String,
    }
    ```
    Place immediately after the `TransferCode` struct definition (around line 703).

    **Update create_transfer_code reducer (SEC-03 for transfer codes):**
    The existing `create_transfer_code` reducer creates a TransferCode row keyed on `code` (primary key). After the table goes private the client cannot read it back. Update `create_transfer_code` to also upsert a `TransferCodeResult` row with `owner = ctx.sender()` and `code = new_code` after inserting the TransferCode row. This means the client can subscribe to `transfer_code_results` and see its own code.

    Also update `use_transfer_code` reducer to delete the TransferCodeResult row for the code's owner when the code is consumed:
    - After deleting from transfer_codes, look up the TransferCode record's owner (save it before deleting) and delete from transfer_code_results by owner.

    Also update `regenerate_recovery_key` and any recovery key creation reducers: after updating recovery_keys, also upsert recovery_code_results for ctx.sender() with the new code.
