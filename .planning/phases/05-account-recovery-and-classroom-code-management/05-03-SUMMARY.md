## Decisions Made

1. **get_class_recovery_codes_reducer.ts already existed**: Codegen had generated it with the correct content. Header comment updated to mark it as hand-written; no content change required.
2. **Download codes button always visible to teachers**: The button is visible whenever `isTeacher && members.length > 0`, not gated on `studentsWithCards > 0`. This allows the initial fetch (which populates the data) rather than requiring codes to already be cached.
3. **300ms setTimeout before handlePrintAll**: The reducer call is async; SpacetimeDB delivers the private table rows via subscription update which triggers a re-render. The delay allows `recoveryKeyByIdentity` to be populated before `handlePrintAll` reads it.
4. **Auto-fixed restore_results gap**: `RestoreResultsRow` import and `restore_results` tablesSchema entry were missing from index.ts. This is a pre-existing bug that would have prevented `useTable(tables.restore_results)` from working in RegisterPage. Fixed inline as Rule 1 (bug fix directly relevant to current task's context in index.ts).
