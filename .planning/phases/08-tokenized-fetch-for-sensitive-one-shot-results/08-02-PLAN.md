```rust
#[reducer]
pub fn use_transfer_code(ctx: &ReducerContext, code: String) -> Result<(), String> {
    let record = ctx.db.transfer_codes().code().find(code.clone())
        .ok_or("Transfer code not found or already used")?;
    ctx.db.transfer_code_results().owner().delete(record.owner);
    ctx.db.transfer_codes().code().delete(code);
    Ok(())
}
```

New reducer to add (simpler — no parameter, no lookup):
```rust
/// CONSUME-01: Delete the restore_results row for the caller after a successful account restore.
/// Idempotent — no error if the row is already gone (identity_disconnected may have cleaned it).
#[reducer]
pub fn consume_restore_result(ctx: &ReducerContext) -> Result<(), String> {
    ctx.db.restore_results().caller().delete(ctx.sender());
    Ok(())
}
```

Placement: Add immediately after the `restore_account` reducer (around line 160). No new imports needed
— `restore_results` table accessor is already imported at the top of security.rs.
